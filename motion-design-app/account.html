This is the account management page, let's list what people can do here:

  • Add a domain where they want to add animations too. We need this later when a request comes from that URL to make sure the account is in good standing (e.g. 'paid'). Even if we don't need this now, I think it is good to start capturing that data. Depending on the account tier an account can have up X domains.
  • For each Domain, URLs (pages) can be added that are being served the animations too. Even if we don't need this now, I think it is good to start capturing that data. Depending on the account tier a domain can have up X URLs.
  • From the URLs people can launch the App to edit the Motions.

In addition we need to 'normal' account stuff like:

  • email address (as the unique identifier)
  • name
  • password
  • account tier (we start with free only)
